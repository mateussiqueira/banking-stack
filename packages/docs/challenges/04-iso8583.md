# 04 — ISO 8583 Simulator

**🇧🇷** Simulador do Padrão ISO 8583 para Mensagens Financeiras  
**🇬🇧** ISO 8583 Financial Message Standard Simulator

---

## Descrição do Desafio

Implementar um simulador do padrão ISO 8583, utilizado em redes de cartões (crédito/débito), ATMs e sistemas de autorização financeira. O ISO 8583 define uma estrutura binária de mensagens com campos bitmap que indicam quais dados estão presentes.

Requisitos:
- Montagem e parsing de mensagens ISO 8583 binárias
- Gerenciamento de bitmaps (primary, secondary)
- Mensagens: Authorization (0100/0110), Financial (0200/0210), Reversal (0420/0430)
- Servidor TCP para comunicação
- Determinação de limites e disponibilidade

---

## Challenge Description

Implement an ISO 8583 standard simulator, used in card networks (credit/debit), ATMs, and financial authorization systems. ISO 8583 defines a binary message structure with bitmap fields indicating which data elements are present.

Requirements:
- Build and parse binary ISO 8583 messages
- Bitmap management (primary, secondary)
- Messages: Authorization (0100/0110), Financial (0200/0210), Reversal (0420/0430)
- TCP server for communication
- Limit and availability determination

## ISO 8583 Message Structure

```
┌─────────┬─────────┬──────────┬──────────┬──────────────────┐
│ MTI     │ Primary │Secondary │  Data     │                  │
│ (4 hex) │ Bitmap  │ Bitmap   │ Elements │                  │
│         │ (8 hex) │ (8 hex)  │ (variable│                  │
│         │         │ (opt)    │  length) │                  │
├─────────┼─────────┼──────────┼──────────┼──────────────────┤
│ 0200    │ F23C... │ (optional│          │                  │
│         │         │  if bit 1│          │                  │
│         │         │  is set) │          │                  │
└─────────┴─────────┴──────────┴──────────┴──────────────────┘
```

### MTI (Message Type Indicator)

| MTI | Meaning | Descrição |
|-----|---------|-----------|
| 0100 | Authorization Request | Solicitação de autorização |
| 0110 | Authorization Response | Resposta de autorização |
| 0200 | Financial Request | Solicitação financeira |
| 0210 | Financial Response | Resposta financeira |
| 0420 | Reversal Request | Solicitação de reversão |
| 0430 | Reversal Response | Resposta de reversão |

### Bitmap Explanation

The bitmap is the heart of ISO 8583. Each bit represents a data element:

```
Primary Bitmap (8 bytes = 64 bits):
Byte 1: Bit 1-8    → Indicates if secondary bitmap exists
Byte 2: Bit 9-16   → Data Element 2 (PAN)
Byte 3: Bit 17-24  → Data Element 3 (Processing Code)
...

Example: Bitmap F23C482000C08004
Binary: 11110010 00111100 01001000 00100000
        00000000 11000000 10000000 00000100
```

## Key Data Elements

| DE | Name | Length | Description |
|----|------|--------|-------------|
| 2 | PAN | Var (max 19) | Primary Account Number |
| 3 | Processing Code | 6 | Transaction type |
| 4 | Amount | 12 | Transaction amount (cents) |
| 7 | Transmission DateTime | 10 | MMDDhhmmss |
| 11 | STAN | 6 | System Trace Audit Number |
| 12 | Local Transaction Time | 6 | HHmmss |
| 13 | Local Transaction Date | 4 | MMDD |
| 37 | Retrieval Reference Number | 12 | Unique reference |
| 38 | Approval Code | 6 | Authorization code |
| 39 | Response Code | 2 | Transaction result |
| 41 | Terminal ID | 8 | POS/ATM identifier |
| 42 | Merchant ID | 15 | Merchant identifier |
| 55 | ICC Data | Var | EMV chip data |

## Code Example: Building an ISO 8583 Message

```typescript
// Building a 0200 (Financial Request) message
function buildFinancialRequest(data: {
  pan: string;
  amount: number;
  terminalId: string;
  merchantId: string;
}): Buffer {
  const mti = Buffer.from('0200', 'ascii');
  
  // Build bitmap based on present fields
  const bitmap = buildBitmap([2, 3, 4, 7, 11, 12, 13, 41, 42]);
  
  // Encode data elements
  const elements = Buffer.concat([
    encodeDE2(data.pan),           // Variable length
    encodeDE3('000000'),           // Purchase
    encodeDE4(data.amount),        // 12 digits
    encodeDE7(getTimestamp()),     // MMDDhhmmss
    encodeDE11(generateSTAN()),    // 6 digits
    encodeDE12(getLocalTime()),    // HHmmss
    encodeDE13(getLocalDate()),    // MMDD
    encodeDE41(data.terminalId),   // 8 chars
    encodeDE42(data.merchantId),   // 15 chars
  ]);
  
  return Buffer.concat([mti, bitmap, elements]);
}

// Parsing an incoming message
function parseMessage(buffer: Buffer): ISO8583Message {
  const mti = buffer.subarray(0, 4).toString('ascii');
  const bitmap = parseBitmap(buffer.subarray(4, 12));
  const elements = parseDataElements(buffer.subarray(12), bitmap);
  
  return { mti, bitmap, elements };
}
```

## Code Example: TCP Server

```typescript
import net from 'net';

const server = net.createServer((socket) => {
  console.log('Client connected');
  
  socket.on('data', (data) => {
    // Parse ISO 8583 message
    const message = parseMessage(data);
    
    // Process based on MTI
    switch (message.mti) {
      case '0100':
        // Authorization request
        const authResponse = processAuthorization(message);
        socket.write(authResponse);
        break;
      case '0200':
        // Financial request
        const finResponse = processFinancial(message);
        socket.write(finResponse);
        break;
      case '0420':
        // Reversal request
        const revResponse = processReversal(message);
        socket.write(revResponse);
        break;
    }
  });
  
  socket.on('end', () => {
    console.log('Client disconnected');
  });
});

server.listen(3004, () => {
  console.log('ISO 8583 server listening on port 3004');
});
```
